# Incident Report – Nmap Reconnaissance Detection

## Incident Summary
An Nmap reconnaissance scan was performed from the Kali Linux attacker machine against the Metasploitable2 target system inside the isolated cybersecurity lab environment.

The activity was monitored using:
- pfSense Firewall
- Snort IDS
- Splunk SIEM
- Wireshark Packet Capture

---

# Environment Information

| Component | IP Address |
|---|---|
| Kali Linux (Attacker) | 192.168.216.128 |
| Metasploitable2 (Victim) | 192.168.216.129 |
| pfSense Firewall | 192.168.216.2 |
| Splunk SIEM | 192.168.216.132 |

---

# Attack Details

## Attack Type
Reconnaissance / Port Scanning

## Tool Used
Nmap

## Command Executed

```bash
nmap -sS -A -T4 192.168.216.129
```

## Attack Objective
The attacker attempted to:
- Discover open ports
- Detect services
- Identify operating system
- Enumerate vulnerable services

---

# Detection & Monitoring

## Snort IDS
Snort detected suspicious scanning activity generated by Nmap reconnaissance traffic.

## pfSense Firewall
pfSense logged network traffic between:
- Source: 192.168.216.128
- Destination: 192.168.216.129

## Splunk SIEM
Splunk successfully received and indexed logs from:
- pfSense
- Snort
- Syslog

Security dashboards were created to visualize:
- Host activity
- Log sources
- Attack timeline
- Firewall activity

# Impact Assessment

The scan successfully identified multiple exposed services on the target system including:
- FTP
- SSH
- Telnet
- SMB
- HTTP
- MySQL
- PostgreSQL

This demonstrates how reconnaissance activity can expose attack surfaces within a network.

---

# Mitigation Recommendations

- Disable unnecessary services
- Block unauthorized scans using firewall rules
- Enable IDS/IPS monitoring
- Use SIEM-based alerting
- Monitor abnormal connection attempts
- Segment critical systems

---

# Conclusion

This incident simulation demonstrated:
- Attack generation

The lab successfully replicated a real-world SOC monitoring workflow within a controlled virtualized environment.
